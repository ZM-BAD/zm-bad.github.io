# ZM-BAD Blog — Project Guide

## Overview

Personal blog (燎原之火) built with Jekyll, deployed to GitHub Pages via GitHub Actions.
URL: https://zmbad.me | Custom domain via CNAME.

## Tech Stack

| Component | Version | Notes |
|-----------|---------|-------|
| Jekyll | 4.4.1 | Static site generator |
| Liquid | 4.0.4 | Template engine |
| Kramdown | 2.5.x | Markdown parser (GFM mode) |
| Rouge | 4.7.x | Syntax highlighting |
| jQuery | 4.0.0 | Loaded in base layout |
| PureCSS | 3.0.0 | Grid framework |
| GLightbox | 3.3.1 | Image lightbox (replaces Fancybox) |
| Ruby | 3.3 (CI) / 4.0 (local) | CI uses 3.3, local any version ≥ 3.3 |

## Commands

```bash
bundle install              # Install dependencies
bundle exec jekyll serve    # Local dev server at http://localhost:4000
bundle exec jekyll build    # Production build to _site/
```

## Project Structure

```
_layouts/base.html          # Main layout (header, sidebar, footer)
_layouts/post.html          # Blog post layout
_layouts/page.html          # Static page layout
_includes/plugins/          # Feature includes (fancybox=GLightbox, baidu_push)
_includes/widgets/          # Sidebar widgets (search, tags, categories, toc)
_posts/                     # Blog posts in Markdown
assets/css/style.css        # Main theme stylesheet (Maupassant v2.0)
assets/js/                  # JavaScript (jquery, glightbox, totop)
_config.yml                 # Site config + Jekyll settings
Gemfile                     # Ruby dependencies
.github/workflows/deploy.yml # CI/CD: build + deploy to GitHub Pages
```

## Key Conventions

- **Asset paths**: Always use `{{ "/assets/..." | prepend: site.baseurl }}` — never hardcode `/assets/...`.
- **Plugins**: Must be listed in BOTH `Gemfile` (under `:jekyll_plugins` group) AND `_config.yml` `plugins:` array.
- **No `github-pages` gem**: Project uses Jekyll 4.x directly with GitHub Actions deployment, not the legacy `github-pages` gem.
- **Comments**: Duoshuo has been removed. If adding comments, use Giscus or Utterances.
- **Search**: Client-side search via `fetch()` against `/assets/data/posts.json`. No jQuery dependency.
- **Back to top**: `totop.js` is vanilla JS, no jQuery dependency.

## Deployment

- **Branch**: `main` triggers auto-deploy via GitHub Actions.
- **Workflow**: `.github/workflows/deploy.yml` — Ruby 3.3 + Jekyll build + Pages deploy.
- **Custom domain**: CNAME file at repo root points to `zmbad.me`.

## Common Gotchas

- Ruby 4.0+ works locally but `github-pages` gem ecosystem doesn't support it yet — that's why we dropped it.
- `sitemap.xml` is auto-generated by `jekyll-sitemap` plugin. Do not create a handwritten one.
- `feed.xml` is auto-generated by `jekyll-feed` plugin.
- The `_includes/plugins/fancybox` file actually loads GLightbox — the filename is kept for backward compatibility with existing post includes.
- Baidu analytics ID lives in `_config.yml` as `baidu_analytics`, rendered inline in `base.html`.
